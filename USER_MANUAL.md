# 用户说明书

_适用于公式识别桌面应用。最后验证日期：2026-06-19。_

---

## 功能概览

本应用是一个常驻后台的公式识别工具。它启动后显示主窗口并驻留系统托盘，可通过主窗口菜单、托盘菜单或全局热键截图识别屏幕上的数学公式，并将结果保存为历史记录。

当前支持：

- 主窗口直接展示历史识别记录
- 新识别记录写入后主窗口会自动刷新
- 历史详情中显示 LaTeX 文本和公式渲染预览
- 使用柔和配色、圆角控件和更舒适的界面间距
- 系统托盘常驻后台运行
- 可设置关闭主窗口时只隐藏窗口、继续后台运行
- 菜单栏直接展示 `截图`、`设置`、`退出`，托盘菜单提供 `主窗口`、`截取公式`、`设置`、`退出`
- 默认全局热键 `Ctrl+Alt+M`
- 多屏幕截图框选，包括外接显示器
- 识别结果以带淡入淡出动画的悬浮框展示，并在 5 秒后自动消失
- 可选自动复制 LaTeX 到剪贴板
- 保存 Markdown 历史记录和可选截图文件
- 使用 PyInstaller 打包生成 Windows exe

## 安装与启动

### 环境要求

| 项目 | 要求 |
| --- | --- |
| 操作系统 | Windows 桌面环境 |
| Python | 建议 Python 3.8 或更高版本 |
| 网络 | 需要能访问配置的 GLM API 地址 |
| API Key | 需要可用的 GLM API Key |

### 从源码启动

```bash
pip install -r requirements.txt
python app.py
```

启动后会显示主窗口，同时创建系统托盘图标。默认关闭主窗口只会隐藏窗口，托盘进程会继续运行；如需退出，请使用菜单中的 `退出`。

## 主窗口

主窗口标题为 `Formula Recognition`，用于浏览历史识别记录。

| 区域 | 作用 |
| --- | --- |
| 历史表格 | 显示时间、状态、置信度和记录文件路径 |
| 详情面板 | 显示选中记录的截图名、错误信息、LaTeX 和公式预览 |
| 菜单栏 | 提供 `截图`、`设置`、`退出` |

右键系统托盘图标也可打开 `主窗口`、`截取公式`、`设置` 和 `退出`。

## 首次配置

首次运行会在 `data/config.json` 创建本地配置文件。推荐通过菜单栏或托盘菜单打开 `设置`。

| 设置项 | 说明 | 示例 |
| --- | --- | --- |
| `API Key` | GLM API Key，必填 | `sk-...` |
| `Endpoint` | GLM API 基础地址 | `https://open.bigmodel.cn/api/paas/v4` |
| `Model` | 使用的模型名称 | `GLM-4V-Flash` |
| `Hotkey` | 全局截图热键 | `ctrl+alt+m` |
| `History Dir` | 历史记录保存目录 | `data/history` |
| 自动复制 | 识别成功后复制 LaTeX 到剪贴板 | 勾选或取消 |
| 保存截图 | 是否保存截图图片 | 勾选或取消 |
| 关闭主窗口时保持后台运行 | 关闭主窗口时只隐藏窗口，继续保留托盘进程 | 默认勾选 |

> 注意：热键修改保存后需要重启应用才会重新注册。API、模型和历史目录会在后续识别时重新读取。

## 截图识别

有三种方式开始截图：

- 主窗口菜单栏点击 `截图`
- 右键托盘图标点击 `截取公式`
- 按全局热键，默认 `Ctrl+Alt+M`

开始截图后，拖动鼠标框选公式区域并松开。识别完成后，LaTeX 会以悬浮框显示并在 5 秒后淡出。如果启用自动复制，结果也会写入系统剪贴板。

## 历史记录

每次识别都会生成 Markdown 历史记录。默认目录为：

```text
data/history/YYYY-MM-DD/
```

记录内容包括时间、状态、截图文件名、置信度、错误信息和 LaTeX。主窗口会读取配置中的历史目录并展示这些记录，详情区域会根据 LaTeX 生成公式预览。每次识别完成并写入记录后，已打开的主窗口会自动刷新。修改 `History Dir` 后，后续截图会写入新目录；重新打开主窗口或点击托盘 `主窗口` 也可刷新显示。

## 打包 exe

开发环境安装打包依赖：

```bash
pip install -r requirements-dev.txt
```

生成 Windows 可执行文件：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/build_exe.ps1
```

输出文件：

```text
dist/FormulaRecognition/FormulaRecognition.exe
```

发布时不要附带包含真实 API Key 的 `data/config.json`。用户首次运行后应自行在设置窗口配置 API。

## 常见问题

### 提示需要填写 api_key

打开 `设置`，填写有效的 `API Key` 后保存，再重新截图。

### 修改热键后没有立即生效

当前版本保存热键后需要重启应用。退出后重新运行 `python app.py` 或重新打开 exe。

### 截图后没有历史图片

检查设置窗口中的 `保存截图到历史记录` 是否已勾选。未勾选时只保存 Markdown 记录。

### 公式预览和原公式不完全一致

当前预览是轻量富文本渲染，支持常见的上标、下标、分式、根号和部分符号。复杂 LaTeX 仍以原始文本为准。

### 托盘图标找不到

检查 Windows 是否把图标折叠到了隐藏托盘区域。

### 关闭主窗口后程序还在运行

这是默认行为。应用会保留托盘进程以便继续使用热键和托盘菜单。可在 `设置` 中取消勾选 `关闭主窗口时保持后台运行`，或直接通过菜单中的 `退出` 结束应用。

## 退出应用

默认情况下，关闭主窗口只是隐藏窗口。使用主窗口菜单栏或托盘菜单中的 `退出` 才会结束应用并注销全局热键。

## 数据与安全

- `data/config.json` 可能包含 API Key，不要提交或发送给他人
- 历史记录可能包含截图和识别结果，请根据内容敏感程度选择保存目录
- 如需清理历史记录，可直接删除配置的历史记录目录


调整整体UI色调为蓝色，并为窗口UI设置合适倒角以保证圆润。
